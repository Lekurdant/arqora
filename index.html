Questions fréquentes

Combien de temps pour un POC ?

Entre 1 et 2 semaines dans 80 % des cas. Le cadrage de discovery permet de fixer ce délai précis en amont. Un POC n'est pas un prototype figé : c'est un livrable fonctionnel, testable par vos utilisateurs.

Qui est propriétaire du code livré ?

Vous. Le code source, les droits et le dépôt Git vous appartiennent intégralement à la fin du projet. La documentation est pensée pour qu'une équipe tech interne puisse reprendre la main sans friction.

Forfait ou régie : comment choisir ?

Forfait si le périmètre est défini (ex : app mobile, automatisation identifiée, refonte site). Régie si vous avez besoin d'un expert IA en renfort ou sur un besoin mouvant. On en discute au 1er appel pour voir ce qui colle le mieux à votre contexte.

Vous travaillez avec quelles technologies ?

Côté IA : Claude (Anthropic), n8n, serveurs MCP custom, OpenAI / Gemini selon les cas. Côté produit : Next.js, React, React Native + Expo, FastAPI (Python), Firestore, Postgres, Redis. Côté infra : VPS avec Docker, Caddy, Cloudflare. Côté e-commerce : Shopify + Storefront API, Hydrogen.

Vous êtes combien sur un projet ?

Je porte chaque projet personnellement — c'est ce qui permet les cycles courts et une vision produit/tech alignée. Pour les projets qui le nécessitent (charge lourde, deadline serrée), je m'entoure d'un réseau de développeurs partenaires de confiance, et je reste le point de contact unique.

Comment démarrer ?

Un premier appel de 30 minutes, offert. On regarde ensemble si votre besoin colle à ce que je sais faire, et s'il y a match, je vous envoie une proposition chiffrée sous 48 h avec roadmap, jalons et livrables.
